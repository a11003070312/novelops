## Task 3: Fix `.claude/settings.json` — hook path matching for absolute paths

**Files:**
- Modify: `.claude/settings.json:12,18`

**Problem:** `CLAUDE_FILE_PATH` is typically an absolute path like
`E:\副业\小说工厂\novelops\chapters\arc-001\chapter-0001.md`.
The regex `chapters[/\\\\]` checks for a relative prefix and will never match
an absolute path. The hook silently does nothing.

**Fix:** Remove the path anchor — match `chapters` and the path separator anywhere
in the string (not just at the start), using a pattern that works on both `/` and `\`.

```json
"command": "bash -c 'set -o pipefail; if echo \"$CLAUDE_FILE_PATH\" | grep -qiE \"[/\\\\\\\\]chapters[/\\\\\\\\].*\\.md$\"; then python scripts/scan-text.py \"$CLAUDE_FILE_PATH\" 2>&1 | tail -80; fi'"
```

```json
"command": "bash -c 'set -o pipefail; if echo \"$CLAUDE_FILE_PATH\" | grep -qiE \"[/\\\\\\\\](state|outline|characters|config|locations)[/\\\\\\\\].*\\.(yaml|md)$\"; then python scripts/check-schema.py \"$CLAUDE_FILE_PATH\" 2>&1 | tail -40; fi'"
```

Note: The change is adding `[/\\\\\\\\]` before `chapters` and before the group,
so the match requires a directory separator before the folder name, preventing
false positives like a path containing "mychapters".

## Task 4: Fix `check-consistency.py` — validate `abandoned_threads` and detect cross-list duplicates

**Files:**
- Modify: `scripts/check-consistency.py` — function `check_plot_thread_timeline` (around line 381)

**Problem A:** `abandoned_threads` is never loaded or validated — schema violations
and cross-list ID collisions go undetected.

**Problem B:** An ID appearing in both `active_threads` and `abandoned_threads`
(or any two lists) is not caught.

**Fix:** Load `abandoned_threads`, validate it the same way as `resolved_threads`,
and add a cross-list duplicate ID check across all three lists.

```python
# After loading active and resolved, add:
abandoned = safe_list(data.get("abandoned_threads"))

# Add abandoned_threads validation (missing abandoned_chapter warning):
for thread in abandoned:
    if not isinstance(thread, dict):
        continue
    tid = thread.get("id", "unknown")
    tname = thread.get("name", tid)
    if "abandoned_chapter" not in thread or thread["abandoned_chapter"] is None:
        issues.append(
            f"已废弃线索 {tid} \"{tname}\" 缺少 abandoned_chapter"
        )

# Cross-list duplicate ID detection:
all_ids: list[str] = []
for thread in active:
    if isinstance(thread, dict) and thread.get("id"):
        all_ids.append(("active", thread["id"], thread.get("name", thread["id"])))
for thread in resolved:
    if isinstance(thread, dict) and thread.get("id"):
        all_ids.append(("resolved", thread["id"], thread.get("name", thread["id"])))
for thread in abandoned:
    if isinstance(thread, dict) and thread.get("id"):
        all_ids.append(("abandoned", thread["id"], thread.get("name", thread["id"])))

id_seen: dict[str, str] = {}
for list_name, tid, tname in all_ids:
    if tid in id_seen:
        issues.append(
            f"线索 ID \"{tid}\" ({tname}) 同时出现在 {id_seen[tid]} 和 {list_name} 列表中"
        )
    else:
        id_seen[tid] = list_name

# Update the total count in the PASS message:
total = len(active) + len(resolved) + len(abandoned)
```

## Task 6: Fix `scan-text.py` — severity fallthrough produces misleading WARN

**Files:**
- Modify: `scripts/scan-text.py` — function `check_banned_patterns` (around line 343)

**Problem:** Any `severity` value other than `"critical"`, `"high"`, or `"medium"`
(e.g., `"low"`, `"info"`) falls through to `else: level = "WARN"` and produces
a warning. This is the wrong default for unknown severity values.

**Fix:** Replace the bare `else` with an explicit `elif severity == "low"` branch
that skips (same as within-threshold medium), and add a final `else` that logs
a WARN about the unknown severity value itself (not the match).

```python
elif severity == "medium":
    continue  # within threshold, skip
elif severity == "low":
    continue  # low severity patterns are informational only
else:
    # Unknown severity value — warn about misconfiguration, not content
    findings.append(
        Finding("WARN", "配置", f"{pat_id} 未知 severity 值: \"{severity}\"")
    )
    continue
```

## Task 7: Fix `check-schema.py` — add `plot-pattern-tracker.yaml` schema validation

**Files:**
- Modify: `scripts/check-schema.py`
  - Add function `validate_plot_pattern_tracker_yaml` (before `discover_files`)
  - Add routing entry in `validate_file` (around line 1051)
  - Add discovery in `discover_files` (around line 1142)

**Problem:** `state/plot-pattern-tracker.yaml` is routed to the generic
`validate_yaml_parseable` which only checks it is a dict. Structural drift
(e.g., `conflict_patterns` becoming a scalar) is not caught.

**Fix — new validator function:**

```python
def validate_plot_pattern_tracker_yaml(data, result: FileResult):
    """Validate state/plot-pattern-tracker.yaml."""
    if not isinstance(data, dict):
        result.fail("文件内容不是有效的 YAML 字典")
        return

    # conflict_patterns must be a list
    if "conflict_patterns" in data:
        if not isinstance(data["conflict_patterns"], list):
            result.fail("conflict_patterns 必须是列表")
        else:
            for idx, entry in enumerate(data["conflict_patterns"]):
                prefix = f"conflict_patterns[{idx}]"
                if not isinstance(entry, dict):
                    result.fail(f"{prefix} 必须是字典")
                    continue
                if "pattern" not in entry:
                    result.fail(f"{prefix} 缺少必填字段: pattern")
                if "occurrences" in entry and not isinstance(entry["occurrences"], list):
                    result.fail(f"{prefix}.occurrences 必须是列表")

    # emotional_beats must be a list
    if "emotional_beats" in data:
        if not isinstance(data["emotional_beats"], list):
            result.fail("emotional_beats 必须是列表")

    # recent_openings must be a list
    if "recent_openings" in data:
        if not isinstance(data["recent_openings"], list):
            result.fail("recent_openings 必须是列表")
        else:
            for idx, entry in enumerate(data["recent_openings"]):
                prefix = f"recent_openings[{idx}]"
                if not isinstance(entry, dict):
                    result.fail(f"{prefix} 必须是字典")
                    continue
                if "chapter" not in entry:
                    result.fail(f"{prefix} 缺少必填字段: chapter")
                elif not isinstance(entry["chapter"], int):
                    result.fail(f"{prefix}.chapter 必须是整数")
                if "type" not in entry:
                    result.fail(f"{prefix} 缺少必填字段: type")

    # consecutive_same_type must be a dict
    if "consecutive_same_type" in data:
        cst = data["consecutive_same_type"]
        if not isinstance(cst, dict):
            result.fail("consecutive_same_type 必须是字典")
        else:
            if "count" in cst and not isinstance(cst["count"], int):
                result.fail("consecutive_same_type.count 必须是整数")

    if not result.has_fail and not result.has_warn:
        result.ok()
```

**Fix — routing (add to `validate_file` before the `state YAML files` elif block):**
